# Java Native Interface (JNI)

Сторона Java:

- определить где будет расположена dll (например на C++)
- загрузить dll
- определить внешний метод (который реализован в dll)
- продумать синхронизацию (например буфер, который парсится на стороне C++)
- вызвать внешний метод

Сторона C:
- получаем заголовочный файл с сигнатурами внешних методов на базе кода java и утилиты javah
- кладем заголовочный файл в проект
- реализовать внешние методы
- наслаждаемся (если запустилось, что вряд ли...)


Передачу аргументов Java <=> C++ реализовываем в канале (разграничиваем обязанности), т.е. основная программа на C++ только через канал отправляет данные, а не из произвольных мест в коде.
