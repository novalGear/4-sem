# KOCEL базы данных

Единый скрипт создания базы.

### Хранение данных в двух схемах:
1) Для работы
2) Транспортная: без зависимостей с минимальной доп. информацией

В транспортной форме нет ID.
ID нужны для связывания таблиц

### Цикл разработки:

- проектирование боевой схемы
- делаем транспортную схему
- перегоняем данные из боевой схемы, восстанавливаем
- если что-то не в порядке, это выявим в этой схеме

### Представление данных (View) (кто это?) и данные:
- в коде используем представления, сами данные - по минимуму.

Денормализация: стараемся не повторять информацию в записях.

Материализованные представления: таблица с явно хранимыми (не вычисляемыми) значениями для быстрых изменений.

### SEQUENCER/TRIGGER:
Sequencer - выдает новый ID
Trigger - добавление нового ключа



Дальнейшее изучение:
Процедурное обеспечение базы

### Trigger

Триггеры бывают на события данных и на события базы (startup, logon пользователя).

Триггер реагирует на событие и вызывает нужную процедуру.

### Глобальные параметры

Делаем таблицу глобальных параметров.
У каждого глобального параметра при изменении выполняется процедура (=> это свойство).
